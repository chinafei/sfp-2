# 预测工程化全流程说明文档 (SFP-2)

本文档说明了从原始数据接入、特征提炼、模型到每天预测结果纯净输出的完整流转过程。一切后续更新与投产部署都已迁移至此工程环境。

## 1. 数据流向与文件夹结构

所有的核心操作均在 `sfp-2/` 目录下完成。
- **`traindata/`** （外部关联）：用于存放省调频的原始表单和 CSV/Excel 源数据。
- **`sfp-2/datasets/npy/`**：存放经过工程化处理洗出的矩阵时序特征 (`.npy`)格式文件（输入矩阵）。
- **`sfp-2/results/`**：存放训练好的模型权重 (`best_dual_tower_model.pt`) 以及纯净版的预测输出日志 (`prediction_only_*.json`)。
- **`sfp-2/scripts/`**：所有的操作脚本入口。

## 2. 预测全流程操作步骤

### 第一步：新增并更新基础数据
将每日最新收集到的业务真实表单数据，拷贝写入到当前项目外部对应的 `traindata/` 原始数据汇总目录内。由于脚本已完全联通至该目录，此步骤完全是普通的数据更新操作不需改代码。

### 第二步：提取生成特征矩阵 (从原始报表到工程 NPY)
每当引入了新的原始台账（例如有未来一天的负荷预测下发），为了把最新日期加入模型的输入数据源中，必须执行打包构建脚本提取所有输入特征向量。
```bash
conda activate tensorflow
python sfp-2/scripts/build_npy_datasets.py
```
> **作用**：此脚本会扫描整个台账范围，重新生成 `sfp-2/datasets/npy/` 中的所有时序与静态特征 (`.npy`)，完美覆盖从历史记录到新填入的所有日期，避免后续前向传播时缺失维度的 NaN。

### 第三步：进行日常预测输出 (纯净预测接口)
若仅需要预测未来或者当天的价格和容量指标（**不依赖该日期的实际出清真值文件参与，也不输出任何对比偏差**），请运行专门剥离出来的纯输出接口脚本 `predict_only.py`，并通过 `--date` 传入执行目标预测的日期。

```bash
conda activate tensorflow
python sfp-2/scripts/predict_only.py --date 2026-03-05
```

**内部执行机制：**
1. 自动寻址加载 `sfp-2/datasets/npy` 中该日期（如 2026-03-05）下的 96点联络线负荷、非市场化出力等综合 12 维特征，和对应日期的静态维特征。
2. 调度调频双塔模型 (`DualTowerBiddingModel`) 加载当前训练出的最优配置节点预测各项基准输出，直接约束各时段 (T1~T5) 的动态价签保护区间。
3. **容量指标校正**：针对调频容量偏离的特性，接口内强制接管了容量修正逻辑，按不同切片将模型粗排容量替代为 `真实负荷曲线均值 × 12.2% (0.122)` 这种符合省调频标杆配置的物理安全算法，确保预测符合现实情况。

**输出成果：**
脚本完成后，会在 `sfp-2/results/` 目录下生成类似 `prediction_only_2026-03-05.json` 的纯预测文件，格式仅存关键输出：
```json
{
    "date": "2026-03-05",
    "segments": {
        "T1": {
            "调频容量需求": 3496.972,
            "边际排序价格": 9.63,
            "市场出清价格(预测均价)": 6.14
        },
        "T2": {
            "调频容量需求": 3475.611,
            "边际排序价格": 9.53,
            "市场出清价格(预测均价)": 5.87
        }
    }
}
```

### 附：随时触发重新训练调优
如果系统运行时积累的新数据周期太长导致当前模型表现出现漂移，一键全自动的寻优执行脚本可帮助随时更迭深度学习网络权重（基于 Optuna 进行 50 epochs 自动调参并选取最佳验证效果模型进行覆盖）：
```bash
python sfp-2/scripts/train_and_tune.py
```
